# Agent 工作规则

## 开发前必读

每次开始开发前，必须先阅读：

- docs/requirements.md
- docs/product-plan.md
- docs/development-steps.md
- 最近一份 dev-logs 开发日志

## 工作方式

1. 不要一次性开发太多功能。
2. 每次只完成一个小模块。
3. 修改代码前，先说明本次计划。
4. 修改完成后，说明改了哪些文件、实现了什么功能。
5. 每次阶段完成后，必须更新 dev-logs。
6. 遇到不确定需求，不要自己乱猜，先提出问题。
7. 项目必须优先适配手机端，同时兼容电脑端。
8. 优先保证项目可以正常运行，再考虑美化和扩展。

## 第一阶段禁止加入

- 支付
- 订单
- 押金托管
- 需求大厅
- 即时聊天
- 复杂后台
- 自动提现
- 原生 iOS / Android App
- 复杂推荐算法
